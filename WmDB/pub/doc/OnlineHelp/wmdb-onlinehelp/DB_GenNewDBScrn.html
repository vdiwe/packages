Screen Descriptions : Generate New Database Service Screen
Generate New Database Service Screen
Screen Heading: WmDB > Service Generation
Use the Generate New Database Service screen to create flow services that access a database.
To access this screen, click Service Generation in the WmDB Navigation panel.
Generate from SQL
Displays the New DB Alias Screen, from which you can generate a database service by specifying a SQL statement.
Generate from table
Displays the Specify Connection Parameters Screen, from which you can begin the process of generating a database service by selecting the table the service is to act on and the database operation the service is to perform.
Choose Data Source
Use this section to select the database that you want the service to access.
Use this field...
To...
Source alias
Specify the database alias for the database you want to use to generate the service. Define database aliases using the Alias Management Screen.
Name the Service
Use this section to name the service and identify the package and folder to which you want to assign the service.
Use this field...
To...
Package
Specify the name of the package to which you want to assign the new service. Select a package from the list.
Folder
Specify the name of the folder to which you want to assign the service. If you want to assign a service to a new folder, type a new folder name using any combination of letters, numbers, or symbols. There is no limit to the number of letters, numbers, or symbols you can use in a folder name, but avoid using punctuation marks such as periods and colons. If you want to assign the service to an existing folder, type the name of that folder exactly as it is defined on the server.
Service
Specify the name that you want to give the service to be generated.
Execute ACL
Specify the ACL group to which you want to limit the service access. Only the users that are in groups that are in the selected ACL list will be able to invoke this service. It is highly recommended that all services have an ACL.
Overwrite Existing
Overwrite an existing service of the same name. If this field is not selected, and a service of the same name exists, then an error is displayed.
If a service of the same name does not exist, the new service will be created.
