Screen Descriptions : Input Binding Generation Screen
Input Binding Generation Screen
Screen Heading: WmDB > Service Generation > Input Binding Generation
Use the Input Binding Generation screen to specify the names and types of input parameters for the database service you are generating.
To access this screen, open the Generate from SQL Screen and click the Evaluate button. The server displays the Input Binding Generation screen, which contains the following:
Generate
Creates the new service and returns to the Generate New Database Service Screen.
Reset
Clears information that you have entered in the fields and refreshes the screen.
Name the Service
Use the fields in this section to name the service and identify the package and folder to which you want to assign the service, if you have not already done so. If you have specified this information on a previous screen, the server displays the information you provided.
Field Name
Description
Package
Specifies the name of the package to which you want to assign the new service. Select a package from the list.
Folder
Specifies the name of the folder to which you want to assign the service. If you want to assign a service to a new folder, type a new folder name using any combination of letters, numbers, or symbols. There is no limit to the number of letters, numbers, or symbols you can use in a folder name, but avoid using punctuation marks such as periods and colons. If you want to assign the service to an existing folder, type the name of that folder exactly as it is defined on the server.
Service
Specifies the name that you want to give the service to be generated.
Execute ACL
Displays the ACL group to which service access is limited. Only the users that are in groups that are in the selected ACL list will be able to invoke this service. It is highly recommended that all services have an ACL. This field is read-only.
Overwrite
Displays the status of the Overwrite function. If the function is On, the creation of a new service will overwrite any existing service of the same name. If the status is Off, and a service of the same name exists, then an error is displayed. This field is read-only.
SQL statement
Displays the SQL statement that the service will perform.
Bind parameters
Use the fields in this section to specify the names and data types for input parameters for your service. If your service requires no input, the server does not display this section of the screen.
Field Name
Description
Parameter #

name: Specifies the name of the input parameter. To set the name, enter characters, numbers, and/or symbols in this field.
type: Specifies the type of the input parameter. To set the type, select the appropriate type from the list.
