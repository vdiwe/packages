Screen Descriptions : New DB Alias Screen
New DB Alias Screen
Screen Heading: WmDB > Alias Management > Add
Use the New DB Alias screen to add a new database alias to the server.
To access this screen, click Alias Management in the WmDB Navigation panel. In the Current Data Sources screen, click the Add button. The server displays the New DB Alias screen, which contains the following:
Submit
Applies the current settings to the server.
Reset
Clears information that you have entered in the fields and refreshes the screen.
Cancel
Exits the screen without creating the database alias and returns you to the Alias Management Screen.
Alias
Specifies the name you want the server to use for the database; for example, Support. You can specify any alias name you want. There is no restriction on the characters you can specify or a limit to the length.
DB URL
Specifies the URL for the database; for example, jdbc:odbc:Support.
DB Username
Specifies the user name that the server must supply to log into the database. If a user name and password are not required, leave this field blank.
DB Password
Specifies the password that the server must supply to log into the database. If a user name and password are not required, leave this field blank.
DB Driver
Specifies the name of the Java class for the JDBC driver. Type the name of the driver in the box. If the server has any drivers currently loaded, it displays the Loaded Drivers field that lists the drivers currently loaded. You can set the DB Driver field by selecting a driver from the Loaded Drivers list.
Important:  
Be sure the server has access to the Java classes for the driver. Typically, you place the Java classes in the server's classpath.
Minimum Connections
The minimum number of connections to pool. The default is 0. If more connections are needed, the server creates additional connections until the value for Maximum Connections, described below, is reached.
Maximum Connections
The maximum number of connections to have in the pool. When the maximum number of connections are open, then the next call to get a connection from the pool will wait until a connection is available (i.e., re-added to the pool). The default is 1.
Expiration Time (ms)
The amount of time to wait before discarding a cached connection from the pool. The default expiration time is 60000 milliseconds.
Loaded Drivers
Specifies the names of all JDBC drivers that the server currently has loaded.
