Specify Connection Parameters Screen
Screen Heading: WmDB > Service Generation > Connection Parameters
Use the Specify Connection Parameters screen to connect to a database. You can specify parameters or restrictions to minimize the scope of your search (i.e., access a few tables instead of all tables in the database).
To access this screen, click Alias Management in the WmDB Navigation panel; the Alias Management Screen will be displayed. Select an alias from the Current Data Sources list, and click the Connect button to display the Specify Connection Parameters screen. Alternatively, click Service Generation in the WmDB Navigation panel; the Generate New Database Service Screen will be displayed. Fill in the fields in the Name the Service section, and click Generate from Table to display the Specify Connection Parameters screen. The Specify Connection Parameters screen contains the following:
Connect
Establishes a connection with the selected database. Clicking this button opens the Tables in (DataSourceName) Screen allowing you to view the tables in the database and to generate a new service using information from a table. To view columns in a specific table, click on the table name.
Reset
Clears information that you have entered in the fields and refreshes the screen.
Cancel
Exits the screen without creating the service and returns you to the original screen.
Data Source
The field in this section displays the database alias for the database that the service accesses.
Field Name
Description
Source Alias
Displays the database alias for the database you want to use to generate the service. Define database aliases using the Alias Management Screen.
Restrictions (optional)
Use the fields in this section to narrow down (or restrict) the list of tables the server displays.
Use this field...
To...
Catalog
Specify the name of the catalog whose tables you want to use. If you are not working with a distributed database, do not restrict by catalog. If you are working with a distributed database, you can specify the name of the database with which you want to work. If you are using DB2, use this field to specify the name of a DB2 location.
Schema Pattern
Specify the schemas whose tables you want to work with (if you are using DB2, use this field to specify an AuthID). If you want tables for all schemas in the selected database, do not restrict by schema pattern. If you want to restrict your search to tables by selected schemas, specify the schema name. You can specify a pattern-matching string if your JDBC supports it. Most drivers support the following pattern-matching characters. However, check your driver's documentation for information about its pattern-matching capabilities.
Char
Description
Example
%
Matches any number of characters
HR% would match: HR3all, HR15mgmt, HR01payroll, etc.
__
Matches a single character
HR3_mgmt would match: HR30mgmt, HR31mgmt, HR3Amgmt, etc.
Table Name Pattern
Specify the names of the tables with which you want to work. Specify a table name or a pattern-matching string that specifies the names of the tables. Most drivers support the following pattern-matching characters. However, check your driver's documentation for information about its pattern-matching capabilities.
Char
Description
Example
%
Matches any number of characters
HR% would match: HR3all, HR15mgmt, HR01payroll, etc.
__
Matches a single character
HR3_mgmt would match: HR30mgmt, HR31mgmt, HR3Amgmt, etc.
Table Types
Specify the types of tables from which you want information. Common JDBC table types include: Table, View, System Table, Alias, and Synonym. Select the table types from the list.
